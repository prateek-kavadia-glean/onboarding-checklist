Buy vs build vs start small

Jason: ownership comes from a metal manufacturing mindset; some ask why pay vs. build a bot in our data center (Mar 27). Response: connectors, permissions, maintenance—and land where pain is acute (support + ops), not IT science projects.

Why buy

Weeks to value with supported connectors—not a multi-year internal R&D program.

Why not only build

Integrations across Zendesk, Microsoft, NetSuite, Slack, Jira, ClickUp are ongoing products—not a script. That is a team, not a weekend project.

Start small

Devon + Jason aligned: minimum value first, expand when credible—same theme as ROI model sensitivity, not a big-bang.

Recommended starting scope

Easy connectors first; custom/legacy after value is proven (Jason + Devon, Mar 27).

Day 1 data sources (typically out-of-the-box)

These are the systems called out as straightforward to connect early:

Source Role in the first chapter
NetSuite Orders, customers, and operational truth for “where is my order?” style questions (within connector capabilities).
Zendesk Tickets, history, and what the front line already works from every day.
Microsoft ecosystem (SharePoint, OneDrive, Outlook) Files, correspondence, and institutional knowledge living alongside formal docs.
Slack Informal resolutions and team context that never made it into a manual.
ClickUp Tasks and internal follow-ups where work is tracked.
Jira Issue tracking where product and support intersect (as appropriate for scope).

Phase 2 / custom integration candidates

Important systems—but do not pay professional services to integrate these on day one unless there is a hard blocker:

  • Dataworks
  • WashLink / vBulletin
  • Other custom, legacy web, or on-premises sources
Defer heavy spend

NAS / on-prem file share: Support is already pushing to get content off local storage because tier-one reps lack access. Connecting NAS through Glean is on the order of ~40 engineering hours—so it belongs in the 50-hour implementation path only if day-one access is truly worth it. On the Mar 27 call, Jason argued it does not make business sense to pay for that integration when the end state is Microsoft cloud anyway—use the project as a forcing function to finish migration, then index where files land.
